Módulo 01

Análisis de Estructuras Web

0:00 – 1:00 60 minutos

Antes de escribir código: ¡Observa!

Antes de crear una página web, los desarrolladores hacen algo que parece simple pero es muy poderoso: observar otras páginas. Abre cualquier sitio web y hazte estas preguntas:

  • ¿Dónde está el logo y el menú de navegación?
  • ¿Dónde está el contenido principal?
  • ¿Hay una barra lateral con información extra?
  • ¿Dónde están los links de redes sociales o el aviso legal?

Notarás que casi todas las páginas web siguen un patrón similar. Eso no es casualidad — es diseño estructural.

La Anatomía de una Página Web

Imagina que una página web es como un periódico físico. Cada zona tiene su propósito:

HTML Antiguo vs HTML5: La Revolución Semántica

¿Sabías que antes todo se hacía con <div>? Antes de HTML5, los desarrolladores usaban etiquetas genéricas para todo. Con HTML5 esto cambió por completo:

✗ Forma antigua — HTML 4 (problemas) 
<div id="cabecera">
  <div id="logo">Mi Sitio</div>
  <div id="menu">Inicio | Blog</div>
</div>

<div id="contenido-principal">
  <div class="articulo">
    <div class="titulo">Mi entrada</div>
    <div class="texto">Contenido...</div>
  </div>
</div>

<div id="barra-lateral">
  <div class="widget">Categorías</div>
</div>

<div id="pie">
  <div class="links">Contacto</div>
</div>
✓ Forma moderna — HTML5 (correcto) 
<header>
  <img src="logo.png" alt="Mi Sitio">
  <nav>Inicio | Blog</nav>
</header>

<main>
  <article>
    <h1>Mi entrada</h1>
    <p>Contenido...</p>
  </article>
</main>

<aside>
  <section>Categorías</section>
</aside>

<footer>
  <p>Contacto</p>
</footer>
💡

Mismo resultado visual, diferente significado

Las dos versiones pueden verse idénticas en el navegador. La diferencia está en lo que el código comunica a buscadores, tecnologías de asistencia y otros desarrolladores.

¿Por qué fallaba la forma antigua?

  • ✗
    Los buscadores no entienden nada. Google no sabe qué parte es el menú y qué parte es contenido importante.
  • ✗
    Las personas con discapacidad visual tampoco. Los lectores de pantalla no pueden navegar bien una página sin semántica.
  • ✗
    Es difícil de mantener. Cuando el código crece, se vuelve un laberinto de <div> anidados sin sentido.
Actividad · 20 min

Desarmando una Página Real

Abre cualquier sitio web en tu navegador (sugerencia: un periódico digital o Wikipedia) y sigue estos pasos:

  1. Haz clic derecho en cualquier parte y selecciona "Inspeccionar"
  2. En el panel, busca las etiquetas: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>
  3. Por cada etiqueta que encuentres: ¿qué contenido tiene? ¿dónde aparece visualmente?
  4. Pregunta: ¿Encontraste alguna etiqueta desconocida? ¿Qué crees que hace?

El Árbol DOM

Cuando escribes HTML, el navegador no lo lee como texto plano. Lo convierte en una estructura llamada DOM (Document Object Model). Piénsalo como un árbol genealógico:

ConceptoDescripciónEjemplo
Elemento padreEl que contiene a otros<nav> es padre de <a>
Elemento hijoEl que está dentro de otro<h1> es hijo de <header>
HermanosComparten el mismo padre<header>, <main> y <footer>
🎯

Dato curioso

Los navegadores usan el árbol DOM para saber qué estilos CSS aplicar y cómo responder a los eventos de JavaScript. ¡Todo parte de un buen HTML semántico!

Módulo 02

Etiquetas Semánticas en Detalle

1:00 – 2:00 60 minutos

El Esqueleto Básico

Antes de hablar de semántica, aquí está la base mínima que siempre debe tener un archivo .html:

index.html — Estructura mínima obligatoria ✓ Correcto 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título de mi página</title>
</head>
<body>
    <!-- Aquí va todo el contenido visible -->
</body>
</html>
Línea¿Qué hace?
<!DOCTYPE html>Le dice al navegador: "esto es HTML5"
<html lang="es">Inicio del documento; lang="es" indica que está en español
<head>Información sobre la página (no se ve en pantalla)
<meta charset="UTF-8">Permite tildes, ñ y caracteres especiales
<meta name="viewport"...>Hace la página responsive (adaptable a móvil)
<title>El texto que aparece en la pestaña del navegador
<body>Todo el contenido que el usuario ve

Etiquetas de Estructura Principal

<header>
Encabezado de la página o sección. Contiene logo, título y navegación.
Puede aparecer más de una vez.
<nav>
Grupo de enlaces de navegación principal del sitio.
Usa aria-label si hay varios.
<main>
Contenido principal único de la página.
Solo 1 por página.
<article>
Contenido independiente que tiene sentido por sí solo.
Post, noticia, comentario.
<section>
Agrupa contenido con un tema en común.
Siempre lleva un título.
<aside>
Contenido complementario, no esencial para el flujo principal.
Sidebar, notas al margen.
<footer>
Pie de página o pie de un artículo. Copyright, links.
Puede aparecer más de una vez.
<figure>
Envuelve imágenes, diagramas o código con su descripción.
Usar con <figcaption>.

Ejemplo completo: <header> y <nav>

header con navegación 
<header>

    <h1>Mi Sitio Web</h1>
    <nav aria-label="Menú principal">
        <ul>
            <li><a href="/">Inicio</a></li>
            <li><a href="/blog">Blog</a></li>
            <li><a href="/contacto">Contacto</a></li>
        </ul>
    </nav>
</header>

article vs section — ¿Cuándo usar cada uno?

Característica<article><section>
¿Tiene sentido solo?✅ Sí, es independienteNo necesariamente
¿Tiene un tema propio?SíSí
¿Necesita título?RecomendadoSí, siempre
EjemplosPost de blog, noticia, tarjeta de productoCapítulo, pestaña, bloque temático
🧠

Truco mental

Si le pondrías un título y agrupa cosas relacionadas → <section>. Si funciona completamente solo fuera de contexto → <article>.

Etiquetas de texto semántico

etiquetas de significado en el texto 
<!-- Tiempo y fechas -->
<time datetime="2024-12-25">Navidad 2024</time>

<!-- Énfasis importante (no solo negrita visual) -->
<strong>¡Atención! Este es un aviso crítico.</strong>

<!-- Énfasis de voz/entonación -->
<em>Este término es especialmente relevante.</em>

<!-- Abreviaciones con explicación al pasar el cursor -->
<abbr title="HyperText Markup Language">HTML</abbr>

<!-- Código dentro de texto -->
<p>Usa la etiqueta <code>&lt;nav&gt;</code> para la navegación.</p>

<!-- Texto resaltado -->
<p>El concepto más importante es <mark>la semántica web</mark>.</p>

<!-- Citas en bloque -->
<blockquote>
    <p>"La web está diseñada para funcionar para todas las personas."</p>
    <footer>— W3C</footer>
</blockquote>

<!-- Figura con descripción -->
<figure>
    <img src="diagrama.png" alt="Diagrama de la estructura HTML5">
    <figcaption>Figura 1: Estructura básica de un documento HTML5</figcaption>
</figure>

¿Por qué importa la Semántica?

¿Por qué molestarse en todo esto si visualmente se ve igual? Hay tres razones poderosas:

01
🔍 SEO — Los buscadores te encuentran
Google, Bing y otros buscadores "leen" tu HTML. Un buen uso de semántica les dice cuál es el título principal, cuál es el artículo importante y cuál es el menú. Resultado: tu página aparece más arriba en los resultados.
02
♿ Accesibilidad — La web para todos
Aproximadamente 1 de cada 6 personas en el mundo vive con algún tipo de discapacidad. Los lectores de pantalla usan las etiquetas semánticas para saltar al contenido principal, anunciar la navegación y listar los artículos. Un código bien escrito puede cambiar la experiencia de alguien.
03
👥 Trabajo en equipo — Código legible
Cuando trabajas en equipo, un código semántico es un código que cualquier compañero puede entender sin explicación. <article> habla por sí mismo. <div class="cosa-principal-2"> no.

Mini-Quiz: ¿Qué etiqueta usarías?

Pon a prueba lo que aprendiste. Haz clic en la etiqueta que usarías en cada situación:

1 El menú con links de "Inicio, Nosotros, Blog, Contacto"
2 Un post completo de blog que se podría compartir solo
3 La sección "Últimas noticias" en la página de un periódico (contiene varios artículos)
4 El pie de página con el copyright y redes sociales
5 Una barra lateral con "Artículos más leídos"
6 El bloque que contiene el logo y el menú superior del sitio
7 El contenido central único de la página (solo puede haber uno)

Referencia

Glosario

Semántica
El significado detrás de las palabras o etiquetas. En HTML, una etiqueta semántica comunica qué es el contenido, no solo cómo se ve.
DOM
Document Object Model. La representación en árbol que hace el navegador del documento HTML. Base para CSS y JavaScript.
Etiqueta
Elemento de HTML encerrado entre < >. Ejemplo: <header>. Pueden ser de apertura, cierre o auto-cerradas.
Atributo
Información adicional dentro de una etiqueta de apertura. Ejemplo: lang="es" en <html lang="es">.
SEO
Search Engine Optimization. Técnicas para aparecer mejor posicionado en buscadores como Google o Bing.
Accesibilidad
Diseño web que permite que personas con discapacidades (visuales, motoras, cognitivas) puedan usar el contenido.
aria-label
Atributo que proporciona una etiqueta descriptiva para tecnologías de asistencia como lectores de pantalla.
<time>
Etiqueta para fechas y horas legibles por máquinas. El atributo datetime usa formato estándar ISO (AAAA-MM-DD).
Responsive
Diseño que se adapta a cualquier tamaño de pantalla: móvil, tablet o computador.
HTML5
La versión actual de HTML, lanzada oficialmente en 2014, que introdujo las etiquetas semánticas y muchas APIs del navegador.
Deprecated
Etiqueta o atributo que ya no se recomienda usar. Ejemplos: <font>, <center>, <b> solo para estilo.
<address>
Etiqueta para información de contacto del autor o de la organización. No es para direcciones físicas genéricas.
